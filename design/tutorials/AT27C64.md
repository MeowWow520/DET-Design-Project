# AT27C64 EPROM 使用教程

## 产品概述

AT27C64 是 Atmel（现 Microchip）生产的 64Kbit（8K × 8bit）紫外线可擦除可编程只读存储器（UV-EPROM），属于 JEDEC 标准 2764 兼容系列。OTP（一次性可编程）版本无需擦除窗口，成本更低。

### 基本参数

| 参数 | 规格 |
|------|------|
| 存储容量 | 64 Kbit = 8 KB（8,192 × 8 bit） |
| 访问时间 | 45ns ~ 200ns（取决于速度等级，如 AT27C64-70 = 70ns） |
| 工作电压 VCC | 5V ±10% |
| 编程电压 VPP | 12.5V（快速编程算法）/ 12.75V |
| 工作电流 | ~30mA（读模式）|
| 待机电流 | ~100µA（CMOS 低功耗） |
| 封装 | 28-pin DIP / 32-pin PLCC / 28-pin TSOP |
| 数据保留 | >10 年 |
| 工艺 | CMOS |

## 引脚定义

### 28-pin DIP 封装引脚图

```
            ┌───── U ─────┐
    VPP ── 1│             │28── VCC
    A12 ── 2│             │27── PGM
     A7 ── 3│             │26── NC
     A6 ── 4│             │25── A8
     A5 ── 5│             │24── A9
     A4 ── 6│             │23── A11
     A3 ── 7│  AT27C64    │22── OE
     A2 ── 8│             │21── A10
     A1 ── 9│             │20── CE
     A0 ──10│             │19── D7
     D0 ──11│             │18── D6
     D1 ──12│             │17── D5
     D2 ──13│             │16── D4
    VSS ──14│             │15── D3
            └─────────────┘
```

### 引脚功能表

| 引脚 | 符号 | 类型 | 功能说明 |
|------|------|------|---------|
| 1 | VPP | 电源 | 编程电压。读模式：接 VCC 或 VIH；编程模式：接 12.5V |
| 2 | A12 | 输入 | 地址线 12（最高位） |
| 3~10 | A7~A0 | 输入 | 地址线 7~0（A0 为最低位） |
| 11~13, 15~19 | D0~D7 | I/O | 8 位数据线。读模式：输出；编程模式：输入 |
| 14 | VSS | 电源 | 地（GND） |
| 20 | CE | 输入 | 片选使能（Chip Enable），**低电平有效**。高电平时芯片进入低功耗待机 |
| 21 | A10 | 输入 | 地址线 10 |
| 22 | OE | 输入 | 输出使能（Output Enable），**低电平有效**。高电平时 D0~D7 呈高阻态 |
| 23 | A11 | 输入 | 地址线 11 |
| 24 | A9 | 输入 | 地址线 9 |
| 25 | A8 | 输入 | 地址线 8 |
| 26 | NC | — | 无连接（No Connect）。在 27128 等更大容量芯片中为 A13 |
| 27 | PGM | 输入 | 编程脉冲（Program），**低电平有效**。读模式：必须接高电平（VCC） |
| 28 | VCC | 电源 | +5V 电源 |

> **地址线总计**：A0~A12 共 **13 根**，可寻址 2¹³ = **8,192 个地址**（8KB）。

### 框图说明

| 模块 | 功能 |
|------|------|
| Address Latch（地址锁存） | 锁存 A0~A12 地址信号，在 CE 下降沿时锁存 |
| X Decoder（行译码） | 将高位地址翻译为存储矩阵的行选通信号 |
| Y Decoder（列译码） | 将低位地址翻译为存储矩阵的列选通信号，选出 8 位数据 |
| Memory Cell Array | 8,192 × 8 个存储单元，浮栅 MOS 管存储 0/1 |
| Control Logic | 根据 CE / OE / PGM / VPP 组合决定工作模式 |
| Output Buffer | 8 位三态输出缓冲器，OE=1 时输出高阻态 |

## 工作模式真值表

| 模式 | CE (20) | OE (22) | PGM (27) | VPP (1) | VCC (28) | D0~D7 |
|------|---------|---------|----------|---------|----------|-------|
| 读 (Read) | L | L | H | VCC | 5V | 数据输出 |
| 输出禁止 (Output Disable) | L | H | H | VCC | 5V | 高阻态 |
| 待机 (Standby) | H | × | × | VCC | 5V | 高阻态 |
| 编程 (Program) | L | H | L (脉冲) | 12.5V | 6V | 数据输入 |
| 编程校验 (Verify) | L | L | H | 12.5V | 6V | 数据输出 |
| 编程禁止 | H | H | × | 12.5V | 6V | 高阻态 |

> L = 低电平（0V），H = 高电平（VCC），× = 任意，Pulse = 低电平脉冲（~100µs）

## 读操作时序

```
   地址 A0~A12 ────X───── 有效地址 ─────X───── 下一地址
                        └───────────┘
   CE (片选)    ──────┐                 ┌─────
                      └─────────────────┘
   OE (输出使能) ────────────┐     ┌───────────
                             └─────┘
   数据 D0~D7   ────────────X  有效数据  X────
                             └──────────┘
                     ← t_ACC →  访问时间
                            ← t_OE →  输出使能时间
```

| 时序参数 | 符号 | AT27C64-70 | 说明 |
|----------|------|------------|------|
| 地址到数据有效 | tACC | ≤70ns | 地址稳定后数据有效时间 |
| CE 到数据有效 | tCE | ≤70ns | 片选有效后数据有效时间 |
| OE 到数据有效 | tOE | ≤35ns | 输出使能后数据有效时间 |
| 地址变化后数据保持 | tOH | ≥0ns | 地址变化后旧数据保持时间 |

> 对于本项目（kHz 级扫描），时序完全不是瓶颈。

## 本项目中的应用

### 仿真接线（读模式）

LED 8×8 点阵驱动建议接线方式：

```
          ┌──────────────────────────┐
          │       AT27C64            │
          │                          │
  计数器 ──┼── A0  (引脚10)           │
  Q0~Q2 ──┼── A1  (引脚9)            │
        ──┼── A2  (引脚8)            │
        ──┼── A3  (引脚7)            │  D0 ──── 74LS595 ──▶ LED点阵列
        ──┼── A4 ~ A12 ── 接 GND     │  ...      或
          │         (只用低3位=8行)   │  D7 ──── 直接驱动 (加限流电阻)
          │                          │
   +5V ───┼── VCC   (引脚28)         │
   +5V ───┼── VPP   (引脚1)          │
   +5V ───┼── PGM   (引脚27)         │
  GND  ───┼── VSS   (引脚14)         │
  GND  ───┼── CE    (引脚20)         │
  GND  ───┼── OE    (引脚22)         │
          │   NC    (引脚26) ── 悬空  │
          └──────────────────────────┘
```

### 地址分配（本项目）

| 地址位 | 用途 | 说明 |
|--------|------|------|
| A0 ~ A2 | 行地址 | 3 位 → 0~7，选择当前扫描行 |
| A3 ~ A12 | 未使用 | 接 GND（只用前 8 字节） |

若需存储多幅图案（轮播动画），可将 A3~A5 接到另一个计数器的输出，在 8 幅图案间切换。

### 图案数据写入示例

每幅图案占用连续的 8 个地址（地址 = 行号），共 8 行 × 8 列 = 8 字节。

```
地址:  0x000  0x001  0x002  0x003  0x004  0x005  0x006  0x007
数据:  0xFF   0x99   0x66   0x42   0x24   0x18   0x00   0x00   ← 爱心
       (行0)  (行1)  (行2)  (行3)  (行4)  (行5)  (行6)  (行7)
```

## Proteus 中使用 AT27C64

### 查找元件

在 Proteus 元件库中搜索关键词：
- `27C64`
- `AT27C64`
- `2764`（JEDEC 标准兼容型号）
- `M2764`（ST 兼容型号）

### 加载图案数据

1. 双击芯片打开属性面板
2. 找到 **Image File** 字段
3. 点击文件夹图标，选择预先生成的 `.BIN` 二进制文件
4. 或在 **Program File** 字段中加载 Intel HEX 格式文件

> 若当前版本 27C64 模型不支持图像加载，尝试换用 `2764` 或 `27256`（地址高位接地即可降级使用）。

### 生成 .BIN 文件的 Python 脚本

```python
# generate_pattern_bin.py
import struct

# 爱心图案，8 行
heart = [0xFF, 0x99, 0x66, 0x42, 0x24, 0x18, 0x00, 0x00]

# 笑脸图案
smile = [0x3C, 0x42, 0xA5, 0x81, 0xA5, 0x99, 0x42, 0x3C]

# 写入 .bin 文件
with open('pattern.bin', 'wb') as f:
    # 第一幅图案（地址 0x000~0x007）
    for byte in heart:
        f.write(struct.pack('B', byte))
    # 第二幅图案（地址 0x008~0x00F）
    for byte in smile:
        f.write(struct.pack('B', byte))

print("pattern.bin 已生成")
```

## 注意事项

1. **未使用引脚**：NC（引脚 26）可以悬空，不用的地址线（如 A3~A12）应接地，避免浮空导致地址不确定。
2. **CE / OE 接地**：在简单纯硬件扫描方案中，CE 和 OE 可以直接接地（始终使能、始终输出），简化控制。输出数据会随地址变化实时更新。
3. **VPP 和 PGM**：读模式下 VPP 必须接 VCC（+5V），PGM 必须接 VCC（+5V），否则芯片不输出数据。
4. **上拉 / 下拉**：不用的地址线（A3~A12）接 GND，确保地址范围在 0x000~0x007（只用前 8 个字节）。
5. **去耦电容**：VCC（28 脚）和 GND（14 脚）之间并联 100nF + 10µF 电容，抑制电源噪声。
6. **输出驱动能力**：EPROM 数据输出驱动能力有限（通常 TTL 电平），驱动 LED 列线时建议加 74LS245 缓冲或 74LS595 锁存。若直驱必须加限流电阻。
7. **Proteus 仿真模型限制**：部分 EPROM 的 Proteus 模型可能默认填充全 0 或全 1，确认方法：用探针监视 D0~D7 输出，看是否按预期变化。
