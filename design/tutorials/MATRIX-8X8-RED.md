# MATRIX-8X8-RED 使用教程

## 产品概述

MATRIX-8X8-RED 是 Waveshare 推出的 8×8 红色 LED 点阵模块，共 64 颗红色 LED 排列成 8 行 × 8 列的矩阵。通过动态扫描方式驱动，可通过 16 个引脚（8 行 + 8 列）控制全部 64 个 LED。

### 基本参数

| 参数 | 规格 |
|------|------|
| LED 数量 | 64 颗 (8×8) |
| LED 颜色 | 红色 |
| LED 直径 | 3mm / 5mm（视具体型号） |
| 正压降 Vf | 1.8V ~ 2.2V |
| 额定电流 If | 10mA ~ 20mA |
| 引脚数量 | 16 脚 |
| 引脚间距 | 2.54mm（标准排针） |
| 模块尺寸 | 约 32mm × 32mm（5mm LED）/ 约 20mm × 20mm（3mm LED） |
| 极性类型 | 共阴 / 共阳（以实际型号标注为准） |

## 工作原理

### 8×8 点阵内部结构

8×8 LED 点阵内部将 64 颗 LED 按矩阵方式连接，每颗 LED 的阳极和阴极分别连接到行线和列线上。根据内部接线方式，分为两种类型：

#### 共阴型 (Common Cathode)

- **行线 (Row)**：连接 LED 的阴极，**低电平**选通行
- **列线 (Col)**：连接 LED 的阳极，**高电平**点亮
- 给行线低电平、列线高电平，对应 LED 亮

```
         C1    C2    C3    ...  C8
          │     │     │           │
    R1 ───├─○──├─○──├─○──       ├─○──  (行1)
    R2 ───├─○──├─○──├─○──       ├─○──  (行2)
    R3 ───├─○──├─○──├─○──       ├─○──  (行3)
    ...   │     │     │           │
    R8 ───└─○──└─○──└─○──       └─○──  (行8)

    行低电平(0)选通 → 列高电平(1)点亮LED
```

#### 共阳型 (Common Anode)

- **行线 (Row)**：连接 LED 的阳极，**高电平**选通行
- **列线 (Col)**：连接 LED 的阴极，**低电平**点亮
- 给行线高电平、列线低电平，对应 LED 亮

> **注意**：使用前务必确认 MATRIX-8X8-RED 的具体极性类型。手册上会有标注。

### 动态扫描原理

8×8=64 个 LED 不可能同时全部点亮（引脚不够），采用**逐行扫描**技术：

```
 ┌─────────┐     ┌──────────┐     ┌──────────┐
 │ 扫描时钟 │────▶│ 行译码器  │────▶│ 行驱动    │────▶ LED 行线
 │ (振荡器) │     │ (3-8译码) │     │ (ULN2803)│     (逐行选通)
 └─────────┘     └──────────┘     └──────────┘

 ┌─────────┐     ┌──────────┐     ┌──────────┐
 │ 图案数据 │────▶│ 锁存器/   │────▶│ 列驱动    │────▶ LED 列线
 │ (ROM等)  │     │ 移位寄存  │     │ (74LS595)│     (列数据)
 └─────────┘     └──────────┘     └──────────┘
```

1. **第 1 行**：行驱动选中第 1 行 → 列驱动输出该行 8 列数据 → 第 1 行 LED 亮
2. **第 2 行**：行驱动选中第 2 行 → 列驱动输出该行 8 列数据 → 第 2 行 LED 亮
3. 依次扫描到第 8 行，再回到第 1 行，周而复始
4. 每行点亮时间极短（<1ms），利用人眼**视觉暂留**效应，看起来所有行同时亮

**扫描频率要求**：
- 每帧 8 行 × 刷新率 > 50Hz
- 行切换频率 > 8 × 50 = 400Hz
- 实际使用时建议 **> 1kHz** 的扫描时钟，以保证无闪烁

## 引脚定义

MATRIX-8X8-RED 模块共有 16 个引脚，分布在模块两侧或下方，各 8 个。

### 典型引脚排列（共阴型）

```
  模块背面（以实际 PCB 丝印为准）

  行控制 (Row)              列控制 (Col)
  ┌──────────────┐          ┌──────────────┐
  │ R1  R2  R3   │          │ C1  C2  C3   │
  │ R4  R5  R6   │          │ C4  C5  C6   │
  │ R7  R8       │          │ C7  C8       │
  └──────────────┘          └──────────────┘
```

| 行引脚 | 功能 | 列引脚 | 功能 |
|--------|------|--------|------|
| R1 | 第 1 行阴极 (Row 1) | C1 | 第 1 列阳极 (Col 1) |
| R2 | 第 2 行阴极 (Row 2) | C2 | 第 2 列阳极 (Col 2) |
| R3 | 第 3 行阴极 (Row 3) | C3 | 第 3 列阳极 (Col 3) |
| R4 | 第 4 行阴极 (Row 4) | C4 | 第 4 列阳极 (Col 4) |
| R5 | 第 5 行阴极 (Row 5) | C5 | 第 5 列阳极 (Col 5) |
| R6 | 第 6 行阴极 (Row 6) | C6 | 第 6 列阳极 (Col 6) |
| R7 | 第 7 行阴极 (Row 7) | C7 | 第 7 列阳极 (Col 7) |
| R8 | 第 8 行阴极 (Row 8) | C8 | 第 8 列阳极 (Col 8) |

> **重要提示**：不同批次的 MATRIX-8X8-RED 引脚排列可能不同，务必参考模块上标注的丝印或数据手册。若为共阳型模块，行引脚为阳极，列引脚为阴极，逻辑电平需反相。

## 图案设计方法

### 像素映射

将 8×8 点阵视为一个 8 像素 × 8 像素的屏幕，每个像素对应一个 LED：

```
  列:  0   1   2   3   4   5   6   7
行0:   [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]
行1:   [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]
行2:   [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]
行3:   [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]
行4:   [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]
行5:   [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]
行6:   [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]
行7:   [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]
```

- `[ ]` = 熄灭 (0)
- `[■]` = 点亮 (1)

### 图案 → 二进制字模

每一行的 8 个 LED 状态用一个 8 位二进制数表示，其中 **bit 0 对应第 0 列（C0），bit 7 对应第 7 列（C7）**（也可约定反序，取决于列驱动接线方式）。

#### 示例：爱心图案

```
  列: C0 C1 C2 C3 C4 C5 C6 C7
行0:  ■  ■  ■  ■  ■  ■  ■  ■    → 0xFF (11111111)
行1:  ■  □  ■  □  □  ■  □  ■    → 0x99 (10011001)
行2:  □  ■  □  ■  ■  □  ■  □    → 0x66 (01100110)
行3:  □  ■  □  □  □  □  ■  □    → 0x42 (01000010)
行4:  □  □  ■  □  □  ■  □  □    → 0x24 (00100100)
行5:  □  □  □  ■  ■  □  □  □    → 0x18 (00011000)
行6:  □  □  □  □  □  □  □  □    → 0x00 (00000000)
行7:  □  □  □  □  □  □  □  □    → 0x00 (00000000)
```

**字模数组**：
```c
unsigned char heart[8] = {
    0xFF, // row 0
    0x99, // row 1
    0x66, // row 2
    0x42, // row 3
    0x24, // row 4
    0x18, // row 5
    0x00, // row 6
    0x00, // row 7
};
```

#### 示例：笑脸图案

```
  列: C0 C1 C2 C3 C4 C5 C6 C7
行0:  □  □  ■  ■  ■  ■  □  □    → 0x3C
行1:  □  ■  □  □  □  □  ■  □    → 0x42
行2:  ■  □  ■  □  □  ■  □  ■    → 0xA5
行3:  ■  □  □  □  □  □  □  ■    → 0x81

行5:  ■  □  □  ■  ■  □  □  ■    → 0x99

行7:  □  □  ■  ■  ■  ■  □  □    → 0x3C
```

## 驱动方案

### 方案一：纯 74 系列分立逻辑驱动（经典数字电路方案）

**适用场景**：纯硬件实现，无需单片机，体现《数字电子技术》课程核心知识。

#### 电路框图

```
┌──────────┐   ┌──────────┐   ┌──────────┐
│ NE555    │──▶│ 74LS161  │──▶│ 74LS138  │──▶ 行驱动 (ULN2803)
│ 多谐振荡  │   │ 4位计数器 │    │ 3-8译码器 │     ↓
└──────────┘   └────┬─────┘   └──────────┘   ┌──────────────┐
                    │                        │ MATRIX-8X8   │
                    │                        │ -RED         │
              ┌─────▼──────┐                 │              │
              │ 图案数据源  │──▶ 74LS595 ──▶   │ 列驱动        │
              │ (ROM/DIP)  │    移位寄存器     └──────────────┘
              └────────────┘
```

**工作流程**：
1. NE555 产生约 1kHz 时钟 → 74LS161 计数 0~7
2. 计数值低 3 位送 74LS138 译码，每次选通 1 行
3. 计数值同时作为图案存储器地址，读出该行的 8 位列数据
4. 列数据经 74LS595 锁存输出到 LED 列线

### 方案二：单片机直驱（动态扫描）

**适用场景**：使用单片机（51/AVR/STM32/Arduino）灵活控制。

#### 接线示例（以 Arduino 为例）

```
MATRIX-8X8-RED (共阴)        Arduino Uno
  行引脚 R1~R8                  D2 ~ D9   (行控制)
  列引脚 C1~C8                  D10 ~ D13, A0~A3 (列控制)

每路串联 220Ω 限流电阻保护 LED
```

#### Arduino 基础扫描代码

```c
// 行控制引脚 (R1~R8) - 共阴型：低电平选中
const int rowPins[8] = {2, 3, 4, 5, 6, 7, 8, 9};
// 列控制引脚 (C1~C8) - 共阴型：高电平点亮
const int colPins[8] = {10, 11, 12, 13, A0, A1, A2, A3};

// 笑脸字模
const byte smile[8] = {
    B00111100, // row 0
    B01000010, // row 1
    B10100101, // row 2
    B10000001, // row 3
    B10100101, // row 4
    B10011001, // row 5
    B01000010, // row 6
    B00111100  // row 7
};

void setup() {
    for (int i = 0; i < 8; i++) {
        pinMode(rowPins[i], OUTPUT);
        pinMode(colPins[i], OUTPUT);
        digitalWrite(rowPins[i], HIGH);  // 共阴型：初始全高（不选通）
        digitalWrite(colPins[i], LOW);   // 共阴型：初始全低（不点亮）
    }
}

void displayPattern(const byte pattern[8], int duration_ms) {
    unsigned long start = millis();
    while (millis() - start < duration_ms) {
        for (int row = 0; row < 8; row++) {
            // 1. 关闭上一行（关闭所有列）
            for (int c = 0; c < 8; c++) {
                digitalWrite(colPins[c], LOW);
            }
            // 2. 选通当前行（共阴型：行低电平）
            digitalWrite(rowPins[row], LOW);

            // 3. 输出当前行的 8 位列数据
            for (int col = 0; col < 8; col++) {
                // bitRead(msb first): col 7→bit7, col 0→bit0
                bool ledOn = bitRead(pattern[row], col);
                digitalWrite(colPins[col], ledOn ? HIGH : LOW);
            }

            delayMicroseconds(800); // 每行点亮时间（8行×800us≈6.4ms/帧 ≈156Hz）

            // 4. 取消选通当前行，准备下一行
            digitalWrite(rowPins[row], HIGH);
        }
    }
}

void loop() {
    displayPattern(smile, 100); // 显示笑脸 100ms
}
```

### 方案三：MAX7219 专用驱动芯片

**适用场景**：快速原型验证，SPI 接口，硬件最简。

#### 接线图

```
MATRIX-8X8-RED      MAX7219 模块       Arduino
  R1~R8 ────────── SEG_A~SEG_G, SEG_DP
  C1~C8 ────────── DIG0~DIG7
                       VCC ────────── 5V
                       GND ────────── GND
                       DIN ────────── D11 (MOSI)
                       CS  ────────── D10 (SS)
                       CLK ────────── D13 (SCK)
```

#### Arduino + MAX7219 代码

```c
#include "LedControl.h"

// DIN→D12, CLK→D11, CS→D10, 设备数=1
LedControl lc = LedControl(12, 11, 10, 1);

byte smile[8] = {
    B00111100, B01000010, B10100101, B10000001,
    B10100101, B10011001, B01000010, B00111100
};

void setup() {
    lc.shutdown(0, false);      // 唤醒 MAX7219
    lc.setIntensity(0, 8);       // 亮度 0~15
    lc.clearDisplay(0);          // 清屏
}

void loop() {
    for (int row = 0; row < 8; row++) {
        lc.setRow(0, row, smile[row]);
    }
    delay(1000);
}
```

**LedControl 库核心函数**：

| 函数 | 说明 |
|------|------|
| `setLed(addr, row, col, state)` | 控制单个 LED：`state=1` 亮, `0` 灭 |
| `setRow(addr, row, value)` | 设置一整行 8 列的亮灭 |
| `setColumn(addr, col, value)` | 设置一整列 8 行的亮灭 |
| `setIntensity(addr, intensity)` | 设置亮度 0~15 |
| `clearDisplay(addr)` | 清屏 |
| `shutdown(addr, status)` | `true`=休眠, `false`=唤醒 |

### 方案四：74LS595 移位寄存器驱动（软硬结合）

**适用场景**：单片机 + 数字芯片混合方案，体现"数字电路"知识同时编程灵活。

```
  单片机 (或硬件控制器)
       │
       ├── 3pin ──▶ 74LS138 ──▶ 行驱动 (ULN2803) ──▶ MATRIX-8X8-RED 行线
       │
       └── 3pin (SPI) ──▶ 74LS595 ──▶ 列驱动 ──▶ MATRIX-8X8-RED 列线
               (串并转换，3 线控制 8 列)
```

## 限流电阻计算

LED 正向电压约 1.8V~2.2V，驱动电压 5V，每路需串联限流电阻：

```
R = (V_drive - V_f) / I_f
  = (5V - 2V) / 10mA
  = 300Ω
```

| 供电电压 | 限流电阻（10mA） | 限流电阻（15mA） | 限流电阻（20mA） |
|----------|------------------|------------------|------------------|
| 5V | 300Ω → 选 330Ω | 200Ω → 选 220Ω | 150Ω → 选 150Ω |
| 3.3V | 130Ω → 选 150Ω | 87Ω → 选 100Ω | 65Ω → 选 68Ω |

> **建议**：使用 220Ω~330Ω，既保证亮度又保护 LED 和驱动芯片。

## 自绘图案生成器

### 在线工具

可以使用以下工具辅助生成字模数据：

- **LED Matrix Editor**（搜索 "8x8 led matrix editor online"）
- 在 8×8 格子上点击设计图案，自动生成 hex/binary 数组

### Python 图案生成脚本

```python
# pattern_generator.py
# 在终端中可视化生成 8x8 图案的字模数据

def draw_pattern(pattern):
    """在终端打印 8x8 图案"""
    for row in pattern:
        line = ""
        for col in range(8):
            line += "■ " if (row >> (7 - col)) & 1 else "□ "
        print(line)

def pattern_to_hex(pattern):
    """将 8 行二进制数据转为十六进制数组"""
    return [f"0x{row:02X}" for row in pattern]

# 示例：爱心图案
heart = [0xFF, 0x99, 0x66, 0x42, 0x24, 0x18, 0x00, 0x00]

print("=== 爱心图案 ===")
draw_pattern(heart)
print(f"\n字模数组: {{ {', '.join(pattern_to_hex(heart))} }}")
```

## 注意事项

1. **确认极性**：使用前用万用表二极管档确认模块是共阴还是共阳，接线错误会导致不亮或全部常亮。
2. **限流电阻不可省略**：每个列驱动引脚（或行驱动引脚）必须串联限流电阻，否则会烧毁 LED。
3. **动态扫描频率**：行扫描频率应 >400Hz（推荐 1kHz），低于此值肉眼会感到明显闪烁。
4. **驱动能力**：74 系列芯片（如 74LS138、74LS595）输出驱动电流有限，行驱动建议加 ULN2803 达林顿管驱动，列驱动可串 74HC245 缓冲。
5. **去耦电容**：模块供电端并联 100nF 瓷片电容 + 10μF 电解电容，减少扫描切换时的电源噪声。
6. **焊接温度**：若需焊接引脚，烙铁温度不超过 300°C，每次焊接时间不超过 3 秒，防止 LED 受热损坏。
