# 74LS138 译码器接线方法

74LS138 是 3-8 线译码器，将 161#1 输出的 3 位行地址（000~111）翻译为 8 条行选通信号（Y0~Y7），每次只选通一行，配合 AT27C64 输出的列数据实现动态扫描。

---

## 引脚定义（16 脚 DIP）

```
         ┌───── U ─────┐
    A  ── 1│            │16── VCC
    B  ── 2│            │15── Y0
    C  ── 3│  74LS138   │14── Y1
  G2A ── 4│            │13── Y2
  G2B ── 5│            │12── Y3
   G1 ── 6│            │11── Y4
   Y7 ── 7│            │10── Y5
  GND ── 8│            │ 9── Y6
         └─────────────┘
```

## 接线方法

### 电源

| 引脚 | 接法 |
|:---:|------|
| 16（VCC） | +5V |
| 8（GND） | GND |

在 16 脚和 8 脚之间就近并联一个 100nF 瓷片去耦电容。

### 使能端

三个使能端必须全部满足条件，芯片才工作。本项目中译码器需要始终使能：

| 引脚 | 符号 | 接法 | 说明 |
|:---:|------|------|------|
| 6 | G1 | +5V | 高有效使能，接高电平 |
| 4 | G2A | GND | 低有效使能，接地 |
| 5 | G2B | GND | 低有效使能，接地 |

> G1 接 +5V，G2A 和 G2B 都接地，三个条件同时满足，芯片永久处于使能状态。

### 地址输入 — 来自 161#1

地址输入端 A、B、C 按照从低位到高位的顺序，接到行扫描计数器（计时模块一）的三个输出：

| 138 引脚 | 符号 | 来源 | 说明 |
|:---:|------|------|------|
| 1 | A（LSB） | 161#1 Q0（14 脚） | 地址最低位 |
| 2 | B | 161#1 Q1（13 脚） | 地址第 1 位 |
| 3 | C（MSB） | 161#1 Q2（12 脚） | 地址最高位 |

地址输入与选中输出的对应关系：
- `A B C = 0 0 0` → Y0 为低电平
- `A B C = 0 0 1` → Y1 为低电平
- `A B C = 0 1 0` → Y2 为低电平
- ...
- `A B C = 1 1 1` → Y7 为低电平

### 输出 — 接 LED 点阵行线

138 的 8 个输出均为**低电平有效**（选中的输出为低，其余为高）。对于共阴型 LED 点阵，行线为阴极，需要**低电平选通**，因此 138 输出可以直接接到行线。

| 138 引脚 | 符号 | 去向 |
|:---:|------|------|
| 15 | Y0 | LED 点阵 行 R1 |
| 14 | Y1 | LED 点阵 行 R2 |
| 13 | Y2 | LED 点阵 行 R3 |
| 12 | Y3 | LED 点阵 行 R4 |
| 11 | Y4 | LED 点阵 行 R5 |
| 10 | Y5 | LED 点阵 行 R6 |
| 9 | Y6 | LED 点阵 行 R7 |
| 7 | Y7 | LED 点阵 行 R8 |

> **注意输出引脚编号不是连续的**：Y7 在 7 脚，紧接着是 GND（8 脚），之后 Y6 在 9 脚（递减排列到 Y0 在 15 脚）。接线时不要按习惯以为 7 脚后面是 Y6。

## 行驱动能力说明

74LS138 每路输出最大灌电流约为 8mA（低电平），拉电流约 0.4mA（高电平）。

一行最多同时点亮 8 个 LED（每列 10mA，共 80mA），**8mA 远远不够**。因此 138 输出不能直接驱动 LED 点阵的行线，必须经过驱动电路。

### 推荐方案：ULN2803 达林顿管阵列

```
138 Y0 ──▶ ULN2803 IN1 ──▶ ULN2803 OUT1 ──▶ LED 行 R1
138 Y1 ──▶ ULN2803 IN2 ──▶ ULN2803 OUT2 ──▶ LED 行 R2
    ...
138 Y7 ──▶ ULN2803 IN8 ──▶ ULN2803 OUT8 ──▶ LED 行 R8
```

ULN2803 每路可灌 **500mA**，完全可以驱动一行中的 8 个 LED（共 80mA）。

**ULN2803 接线**：
- 1~8 脚（IN1~IN8）：接 138 的 Y0~Y7
- 11~18 脚（OUT1~OUT8）：接 LED 点阵行线 R1~R8
- 9 脚（COM）：接 +5V（内部续流二极管公共端，驱动 LED 时不可省略）
- 10 脚（GND）：接地

> **注意**：ULN2803 是反相的。输入高 → 输出对地导通（低电平）。而 138 输出是低有效（选中=低），所以需要一级反相。但实际上 138 选中时输出低 → 经过 ULN2803 → 输出高阻（不导通），这就反了。
>
> **正确理解**：138 输出低（选中行）→ ULN2803 关断 → 行线靠上拉电阻拉高 → 行为高 → LED 行不导通（共阴型行是阴极，高不亮）。这就错了。
>
> 所以对于共阴型点阵，138 输出需要的是**被选中行低电平**。有几种解法：
>
> **方法 A（最简单）**：在 138 输出端接上拉电阻直接驱动，不经过 ULN2803。适合 Proteus 仿真，实物可能不够亮。
>
> **方法 B**：用 74LS245 总线缓冲器，138 输出接 245 输入，245 输出接行线。245 每路可灌约 24mA。
>
> **方法 C**：使用共阳型 LED 点阵。此时行线是阳极（需高选通），列线是阴极（低点亮）。138 输出（低选中）反相后驱动行。或者用 74LS138 直接选通 PNP 晶体管。
>
> 如果你使用的是 **MATRIX-8X8-RED 共阴型**，建议采用 74LS245 缓冲方案，或直接 138 接行线（Proteus 仿真中电流不限，可以直接用）。

## Proteus 仿真注意事项

- Proteus 中 138 输出电流不受限，可直接接 LED 点阵行线（仿真无需 ULN2803）
- 地址输入端（A/B/C）若悬空会被视为低电平，导致 Y0 始终选中。确保 161#1 确实有输出
- 使能端 G1、G2A、G2B 在 Proteus 中同样需要正确接线，否则所有输出全高
- 用逻辑分析仪或探针监视地址和 Y0~Y7，确认译码结果是否正确

## 完整接线总览

```
          161#1 (行扫描计数器)
          ├─ Q0 ── 138 A(1)
          ├─ Q1 ── 138 B(2)
          └─ Q2 ── 138 C(3)

          +5V ── 138 G1(6)           138 Y0(15) ── LED R1
          GND ── 138 G2A(4)          138 Y1(14) ── LED R2
          GND ── 138 G2B(5)          138 Y2(13) ── LED R3
          +5V ── 138 VCC(16)         138 Y3(12) ── LED R4
          GND ── 138 GND(8)          138 Y4(11) ── LED R5
                                      138 Y5(10) ── LED R6
                                      138 Y6(9)  ── LED R7
                                      138 Y7(7)  ── LED R8
```
