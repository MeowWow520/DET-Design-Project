# LED 8×8 点阵驱动电路 — 设计方案汇总

## 项目需求回顾

利用数字电子技术，使用 8×8 LED 灯板，个性化显示图案。

---

## 设计要点分析

驱动 8×8 LED 点阵的核心问题：

| 问题 | 说明 |
|------|------|
| **引脚数限制** | 8×8=64 个 LED，若静态驱动需 64+ 引脚；动态扫描只需 8（行）+ 8（列）= 16 个 I/O |
| **动态扫描** | 逐行（或逐列）快速轮流点亮，利用人眼视觉暂留形成完整图案 |
| **行选通** | 每次只选通一行（或一列），需译码器或环形计数器 |
| **列数据** | 当前被选通行上哪些 LED 亮，需锁存器 / 移位寄存器 / ROM 提供数据 |
| **扫描时钟** | 需足够快的扫描频率（>50Hz×8=400Hz 以上），由振荡器提供 |
| **图案存储** | 图案数据存放方式：二极管矩阵、ROM/EEPROM、DIP 开关、或程序代码 |

---

## 方案一：纯 74 系列分立逻辑方案（最经典）

### 思路

完全使用 74 系列标准数字逻辑芯片搭建，不涉及任何可编程器件。这是与《数字电子技术》课本结合最紧密的方案，几乎覆盖课程所有核心知识点。

### 系统框图

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 555 定时器 │──▶│ 74LS161  │──▶│ 74LS138  │──▶│ 8×8 LED  │
│ (多谐振荡) │   │ 计数器   │   │ 3-8 译码器│   │ 点阵(行)  │
└──────────┘   └────┬─────┘   └──────────┘   └──────────┘
                    │
                    ▼
             ┌──────────┐   ┌──────────┐
             │ 74LS595×2│──▶│ 8×8 LED  │
             │ 移位寄存器│   │ 点阵(列)  │
             └────┬─────┘   └──────────┘
                  │
             ┌────▼─────┐
             │ 图案存储  │
             │ (二极管/  │
             │  DIP开关) │
             └──────────┘
```

### 所需核心芯片

| 芯片 | 功能 | 对应课程知识点 |
|------|------|---------------|
| NE555 | 多谐振荡器，产生扫描时钟 | 脉冲产生与整形 |
| 74LS161 | 4 位二进制同步计数器，提供地址 | 计数器 |
| 74LS138 | 3-8 线译码器，行选通 | 译码器 |
| 74LS595 | 8 位移位寄存器（串入并出），列数据 | 移位寄存器 |
| 74LS373/573 | 8 位锁存器（可选，稳定显示） | 锁存器 |
| 1N4148×64 | 二极管矩阵编码图案 | 组合逻辑 / 编码 |

### 工作原理

1. 555 产生约 1kHz 方波时钟；
2. 74LS161 对时钟计数，输出 4 位地址（实际用 3 位即可对应 8 行）；
3. 低 3 位送 74LS138 译码，每次选通 1 行（8 行轮流）；
4. 计数器的 3 位同时作为图案存储（如二极管矩阵或小容量 ROM）的地址；
5. 图案存储输出 8 位列数据，经 74LS595 锁存后驱动 LED 列线；
6. 扫完 8 行后重新开始，形成完整画面。

### 优缺点

| 优点 | 缺点 |
|------|------|
| 与课程知识结合最紧密 | 芯片数量多，布线复杂 |
| 纯硬件实现，实时性强 | 修改图案需改变硬件连线 |
| 可充分展示数字电路设计能力 | 灵活性低 |

## 方案二：ROM/EEPROM 存储 + 计数器扫描方案

### 思路

将图案数据预先写入 ROM / EEPROM（如 28C64、AT28C16 等），用计数器产生地址依次读出，经锁存器驱动 LED 点阵。此方案在方案一基础上，用存储器替代二极管矩阵，图案可重写。

### 系统框图

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 时钟源   │──▶│ 计数器   │──▶│ 地址总线 │──▶│ EEPROM   │
│(555/晶振)│   │(74LS161) │   │ A0-A5    │   │ 图案存储 │
└──────────┘   └──────────┘   └──────────┘   └────┬─────┘
                                                  │
      ┌──────────┐   ┌──────────┐                │ 数据 D0-D7
      │ 行驱动   │◀──│ 3-8译码器│◀─── 地址高3位 ──┘
      │(ULN2803)│   │(74LS138)│
      └────┬─────┘   └──────────┘
           ▼
      ┌──────────┐
      │ 8×8 LED  │
      │ 点阵     │
      └──────────┘
```

### 关键技术点

- **地址分配**：A0–A2 选行（0–7），A3–A5 选图案页（可有 8 种不同图案）
- **数据位**：D0–D7 对应 8 列
- **图案编程**：使用编程器（如 TL866）将图案数据烧录到 EEPROM
- **动画效果**：可通过切换高位地址线实现多帧图案轮播

### 优缺点

| 优点 | 缺点 |
|------|------|
| 图案可重写，灵活度提升 | 需额外的编程器设备 |
| 仍是纯硬件方案 | 存储容量利用不充分（8B/图案） |
| 可存储多幅图案轮播 | — |

## 方案三：FPGA / CPLD 可编程逻辑方案

### 思路

将所有数字逻辑（计数器、译码器、移位寄存器、图案 ROM）全部集成到一片 FPGA 或 CPLD 中，用 Verilog / VHDL 硬件描述语言设计。外部只需 LED 点阵 + 限流电阻 + 时钟源（或使用片内振荡器）。

### 系统框图

```
┌─────────────────────────────┐
│        FPGA / CPLD           │
│                              │
│  ┌────────┐  ┌────────────┐ │      ┌──────────┐
│  │ 时钟   │  │ 图案数据   │ │      │ 8×8 LED  │
│  │ 分频器 │  │ ROM (LUT)  │ │      │ 点阵     │
│  └────┬───┘  └─────┬──────┘ │      └──────────┘
│       │            │         │
│  ┌────▼───┐  ┌────▼──────┐  │
│  │ 计数器 │  │ 移位寄存器 │──┼──────▶ 列 (8)
│  └────┬───┘  │ / 锁存器   │  │
│       │      └────────────┘  │
│  ┌────▼───┐                  │
│  │ 3-8    │──────────────────┼──────▶ 行 (8)
│  │ 译码器 │                  │
│  └────────┘                  │
└─────────────────────────────┘
```

### 推荐平台

- **Xilinx**：Spartan-6 / Artix-7 开发板，用 Vivado + Verilog
- **Intel (Altera)**：EP4CE6 / MAX10 开发板，用 Quartus + Verilog
- **国产**：高云半导体 GW1N 系列（云源软件）或安路科技

### 关键技术点

- 顶层模块实例化子模块（计数器、译码器、ROM、扫描控制）
- 图案数据存入片上 Block RAM / 分布式 RAM（用 `initial` 或 `.coe`/`.mif` 文件初始化）
- 按键切换图案（加一个按键消抖模块 + 图案索引计数器）
- 可扩展为 PWM 调光（控制每行点亮占空比）

### 优缺点

| 优点 | 缺点 |
|------|------|
| 芯片数量最少，体积小 | 需掌握 Verilog/VHDL |
| 灵活度最高，可软件修改一切 | 开发环境搭建较复杂 |
| 与课程 PLD 章节对应 | 开发板成本相对较高 |

## 方案四：单片机 + 74 系列混合方案

### 思路

用单片机（STC89C52、ATmega328P、STM32 等）作为控制核心，软件完成图案存储和扫描时序控制；行选通和列驱动仍使用 74 系列芯片（74LS138 + 74LS595 / 74HC573），在"软硬结合"中体会数字电路原理。

### 系统框图

```
┌──────────────┐
│  单片机       │
│ (51/AVR/STM) │
│              │
│  GPIO ───────┼──▶ 74LS138 ──▶ 行驱动 ──▶ LED 点阵 (行)
│  SPI/GPIO ───┼──▶ 74LS595 ──▶ 列驱动 ──▶ LED 点阵 (列)
│  中断/定时器 │
└──────────────┘
```

### 关键技术点

- 单片机定时器中断产生扫描节拍（如 1ms 一行，8ms 一帧 = 125Hz 刷新率）
- 字模数据存入单片机 Flash（`const unsigned char` 数组）
- 串行口（SPI 或模拟 SPI）向 74LS595 发送列数据
- 多图案可通过按键切换，或在代码中预设多帧动画
- 可选：串口通信，上位机实时修改图案

### 优缺点

| 优点 | 缺点 |
|------|------|
| 图案修改最方便（改代码即可） | 数字电路硬件占比降低 |
| 可扩展性强（加传感器、通信等） | 需学习单片机编程 |
| 当前工程实践中最常用方案 | "纯数字电路"属性偏弱 |

## 方案五：专用 LED 驱动芯片方案

### 思路

使用专用 LED 点阵驱动芯片（如 MAX7219、TM1640、HT16K33），内部已集成扫描控制、译码、恒流驱动等全部逻辑，外部 MCU 仅需通过 SPI/I²C 发送命令即可。重点在于理解芯片内部电路框图，而非自行搭建。

### 典型芯片对比

| 芯片 | 接口 | 驱动能力 | 特点 |
|------|------|---------|------|
| MAX7219 | SPI (3 线) | 8 位数码管 / 8×8 点阵 | 经典芯片，内部自带 BCD 译码 |
| TM1640 | 2 线串行 | 16×8 段 / 8×8 点阵 | 国产低成本替代 |
| HT16K33 | I²C | 16×8 / 8×8 点阵 | 内置按键扫描 |

### 系统框图

```
┌──────────┐    SPI/I²C    ┌──────────────┐    ┌──────────┐
│ 单片机    │◀─────────────▶│ MAX7219      │───▶│ 8×8 LED  │
│ (任意MCU) │               │ / TM1640     │    │ 点阵     │
└──────────┘               └──────────────┘    └──────────┘
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 硬件最简单，几根线搞定 | 数字电路设计含量最低 |
| 可靠性高，芯片内建保护 | 不体现计数器、译码器等基础知识 |
| 适合快速原型验证 | 可能不符合课程"数字电路"要求 |

> **提示**：此方案适合作为"对比方案"在报告中分析，但不建议作为主设计方案，因为数字电路知识点覆盖太少。

## 方案六：DIP 开关可配置 + 手动扫描方案

### 思路

用 8 个 DIP 拨码开关设置当前行的列数据，用户手动按下按键逐行切换扫描，适合课堂演示逐行扫描的原理。可扩展：在两个模式间切换——「手动逐行演示」和「自动扫描显示」。

### 系统框图

```
┌──────────┐   ┌──────────┐   ┌──────────┐
│ DIP 开关 │──▶│ 三态缓冲 │──▶│ 列驱动   │──▶ LED 点阵(列)
│ (8位)    │   │ 74LS244  │   │          │
└──────────┘   └──────────┘   └──────────┘
                                  ▲
┌──────────┐   ┌──────────┐      │
│ 按键/     │──▶│ 74LS161  │──┐   │
│ 555时钟   │   │ 计数器   │  │   │
└──────────┘   └──────────┘  │   │
                             ▼   │
                        ┌──────────┐
                        │ 74LS138  │──▶ 行驱动 ──▶ LED 点阵(行)
                        └──────────┘
```

### 工作模式

- **手动模式**：按一次按键 → 计数器 +1 → 切换到下一行 → 观察该行显示内容
- **自动模式**：555 提供连续时钟 → 8 行快速扫描 → 视觉暂留显示完整图案
- **图案修改**：改变 DIP 开关位置即可实时改变图案

### 优缺点

| 优点 | 缺点 |
|------|------|
| 直观展示动态扫描原理 | 手动模式下看不到完整图案 |
| 可实时调整图案（拨码开关） | DIP 开关占用体积大 |
| 教学演示效果好 | — |

## 建议

1. **如果课程偏重传统数字电路实验**（面包板搭电路、74 系列芯片为主），推荐 **方案一** 或 **方案二** 作为主方案。
2. **如果有 FPGA 开发板可用** 且课程涉及 Verilog/VHDL，推荐 **方案三**。
3. **如果允许单片机** 且时间紧张，推荐 **方案四**，可在报告中重点分析 74 系列驱动部分。
4. **方案六** 适合在答辩/展示环节作为原理演示的辅助设计。
5. 实际实践中，可取某一方案为主体，结合其他方案的思路进行优化。

---

*以上方案均为概要思路，具体实现需结合元器件选型、电路仿真（如 Proteus / Multisim）、原理图和 PCB 设计等后续步骤完成。*
